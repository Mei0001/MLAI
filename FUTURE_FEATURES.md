## 🎯 プロジェクト概要

MLAIは、個人用のAI音声アシスタントプラットフォームを目指すプロジェクトです。音声の録音、文字起こし、AI要約、検索などの機能を統合し、日常の会議やメモを効率的に管理します。

---

## 🔧 コア機能

### 1. 音声録音機能

| 項目 | 詳細 |
|------|------|
| リアルタイム録音 | マイク入力からの音声をリアルタイムで録音 |
| システム音声キャプチャ | PC上で再生される音声の録音（オンライン会議対応） |
| ファイルインポート | 既存の音声ファイル（MP3, WAV, M4A等）のインポート |
| バックグラウンド録音 | アプリがバックグラウンドでも録音継続 |
| 自動分割 | 長時間録音の自動チャプター分割 |

### 2. 音声認識（Speech-to-Text）

| 項目 | 詳細 |
|------|------|
| リアルタイム文字起こし | 録音と同時にテキスト化 |
| 多言語対応 | 日本語、英語、その他主要言語のサポート |
| 話者分離（Speaker Diarization） | 複数話者の識別と分離 |
| 句読点自動挿入 | 自然な句読点の自動配置 |
| カスタム辞書 | 専門用語や固有名詞の登録 |

### 3. AI要約・分析機能

| 項目 | 詳細 |
|------|------|
| 自動要約生成 | 会議内容の要約を自動生成 |
| アクションアイテム抽出 | TODOや次のアクションの自動抽出 |
| キーポイント抽出 | 重要なポイントのハイライト |
| 質疑応答 | 録音内容に関する質問への回答 |
| 感情分析 | 会話のトーンや感情の分析 |

### 4. データ管理・検索

| 項目 | 詳細 |
|------|------|
| 全文検索 | 文字起こしテキストの全文検索 |
| タグ・カテゴリ管理 | 録音のタグ付けと分類 |
| タイムスタンプ検索 | 特定の発言箇所へのジャンプ |
| フィルタリング | 日付、話者、キーワードでのフィルタ |

## ⚙️ 技術要件

### バックエンド

```
- 言語: Python / TypeScript (Node.js)
- フレームワーク: FastAPI / NestJS
- データベース: PostgreSQL + pgvector（ベクトル検索用）
- キャッシュ: Redis
- メッセージキュー: Celery / BullMQ（非同期処理用）
```

### フロントエンド

```
- フレームワーク: Next.js / React
- 状態管理: Zustand / Jotai
- UI: Tailwind CSS + shadcn/ui
- 音声処理: Web Audio API
```

### AI/ML

```
- 音声認識: Whisper (OpenAI) / Google Speech-to-Text
- LLM: OpenAI GPT-4 / Claude / ローカルLLM (Ollama)
- 話者分離: pyannote.audio
- 埋め込み: text-embedding-ada-002 / sentence-transformers
```

### インフラ

```
- クラウド: AWS / GCP / Azure
- コンテナ: Docker + Kubernetes
- CI/CD: GitHub Actions
- モニタリング: Prometheus + Grafana
```

## 📊 非機能要件

### パフォーマンス
| 要件 | 目標値 |
|------|--------|
| 文字起こし遅延 | < 3秒（リアルタイム） |
| 要約生成時間 | < 30秒（1時間の録音） |
| 検索レスポンス | < 500ms |
| 同時接続数 | 1000+ |
